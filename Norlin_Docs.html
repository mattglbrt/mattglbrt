Getting Started

To run the theme locally, navigate to the theme directory and follow these steps:

Prerequisites: Make sure you have Node.js (v18 or higher) and npm installed on your system.

1. Install dependencies:

npm install

2. Start the development server:

npm run dev

3. Build for production:

npm run build

4. Preview production build:

npm run preview

The development server will typically start at http://localhost:4321

Theme Configurations

All theme settings are centralized in the src/data/settings.ts file. This file contains all the customization options for your Astro theme.

Navigation

Configure your site navigation menu in src/data/settings.ts:

navigation: {
  items: [
    { title: 'Home', url: '/' },
    { title: 'About', url: '/about/' },
    { title: 'Contact Me', url: '/contact/' },
    { title: 'Style Guide', url: '/styleguide/' },
    { 
      title: 'Get Theme', 
      url: 'http://anvodstudio.com/themes/astro/norlin', 
      external: true 
    },
  ],
}

Navigation Item Properties:

  • title: Menu item text
  • url: Link destination
  • external: Set to true for external links (optional)

Important: Always include a trailing slash (/) at the end of internal links for proper routing in Astro.

Hero Section

Customize the hero section that appears on your homepage:

Posts

Create blog posts in the src/content/posts/ directory. Each post is a markdown file (.md) or MDX file (.mdx).

Post Frontmatter

Each post should include frontmatter with the following fields:

---
title: "Your Post Title"
description: "A brief description of your post"
image: "../../images/post-image.jpg"
tags: [Work, Notes]
featured: true
date: 2025-01-15
---

Frontmatter Fields:

  • title: Post title (required)
  • description: Post description for SEO and previews
  • image: Featured image path (relative to src/)
  • tags: Array of tags (use dashes instead of spaces: web-development)
  • featured: Set to true to mark as featured post (displays star icon ‚òÜ)
  • date: Publication date (YYYY-MM-DD format)
Pagination Settings

Control how posts are displayed on the homepage:

pagination: {
  initial: 8, // Number of posts to show initially
  step: 3     // Number of posts to load with "Load More"
}
Wide Images

To create full-width images in your posts, use the :::wide directive:

:::wide
![Image description](../../images/image.jpg)
*Photo caption*
:::
Image Galleries

Create image galleries using the :::gallery directive:

:::gallery
  ![Abstract](../../images/501.jpg)
  ![Bike](../../images/901.jpg)
  ![Man](../../images/509.jpg)
  *Gallery / Unsplash*
:::

Tip: Store all post images in the src/images/ directory for easy management.

Newsletter

Enable and configure the newsletter subscription form:

newsletter: {
  enabled: true,
  mailchimpIdentifier: 'gmail.us21.list-manage.com/subscribe/post?u=...',
  description: 'Subscribe to our newsletter...',
  buttonText: 'Join',
}

Newsletter Properties:

  • enabled: Set to true to display newsletter form
  • mailchimpIdentifier: Your Mailchimp form action URL
  • description: Text displayed above the form
  • buttonText: Submit button text

To get your Mailchimp identifier, create a form in your Mailchimp account and copy the form action URL.

Contact Form

Configure the contact form using Formspree:

contact: {
  email: 'your_formspree_form_id', // Example: 'mbjebqko'
}

To set up the contact form:

  1. Go to Formspree.io and create a free account
  2. Create a new form and get your form ID
  3. Add the form ID to the email field in settings

Comments (Disqus)

Enable Disqus comments on your blog posts:

comments: {
  enabled: true,
  disqusIdentifier: 'anvodstudio-demo',
}

To set up Disqus:

  1. Create a Disqus account at disqus.com
  2. Register your website and get your Disqus shortname
  3. Replace anvodstudio-demo with your Disqus shortname
  4. Set enabled: true to display comments

If comments fail to load, make sure you have registered your website with Disqus.

Google Analytics

Add Google Analytics tracking to your site:

analytics: {
  google: '', // GA4 Measurement ID
}

To enable Google Analytics:

  1. Create a Google Analytics 4 property
  2. Get your Measurement ID (format: G-XXXXXXXXXX)
  3. Add it to the google field in settings

Deployment

Astro provides multiple deployment options. Here are the most popular:

Deploy to Netlify
  1. Push your project to GitHub
  2. Connect your repository to Netlify
  3. Set build command: npm run build
  4. Set publish directory: dist
Deploy to Vercel
  1. Push your project to GitHub
  2. Import your repository to Vercel
  3. Vercel will auto-detect Astro and configure settings
Deploy to GitHub Pages
  1. Update src/data/settings.ts with your site URL
  2. Use the GitHub Actions workflow for automatic deployment

For detailed deployment instructions, check the Astro Deployment Guide.

Styling & Themes

The theme uses SCSS for styling. All styles are organized in src/styles/:

  • 0-settings/ ‚Äì Variables, colors, and global settings
  • 1-tools/ ‚Äì Functions and resets
  • 2-base/ ‚Äì Base styles
  • 3-modules/ ‚Äì Component-specific styles
  • 4-layouts/ ‚Äì Layout styles
  • main.scss ‚Äì Main stylesheet that imports all modules
Customizing Colors

Edit color variables in src/styles/0-settings/_variables.scss to customize the theme colors and create your own color scheme.

Dark Mode

The theme is designed with a dark color scheme by default. All colors and contrasts are optimized for dark mode viewing experience.
